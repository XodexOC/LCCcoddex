# [05] Режимы адресации

> **Трек:** Assembly · **Уровень:** 05 · **Сложность:** ★★★☆☆

## 1. Какую проблему решаем?

В прошлом уроке мы научились читать и писать в память:

```assembly
mov rax, [my_var]    ; читаем значение по адресу my_var
mov [my_var], rax    ; пишем по адресу my_var
```

Это работает, но адрес всегда жёстко задан. А что, если мы хотим:

1. **Пройти по массиву циклом** — адрес меняется (arr[0], arr[1], arr[2]...)
2. **Обратиться к элементу по индексу** — индекс лежит в регистре (rsi = 3, нужен arr[3])
3. **Обратиться к полю структуры** — нужен адрес = база + смещение
4. **Вычислить адрес, не читая память** — просто получить адрес как число

Для всего этого существуют разные **режимы адресации** (addressing modes) — способы указать процессору, по какому адресу обратиться к памяти.

## 2. Суть

У процессора x86-64 есть несколько способов вычислить адрес памяти:

```
Форма                    Пример                     Что делает
──────────────────────   ─────────────────────      ─────────────────────
Direct                   [addr]                     Адрес задан меткой
Register indirect        [rax]                      Адрес в регистре
Base + displacement      [rax + 10]                 Регистр + число
Indexed                  [arr + rsi*4]              Метка + индекс * размер
Base + index*scale       [rbx + rsi*4]              База + индекс * размер
Full                     [rbx + rsi*4 + 10]         Всё вместе
```

Общая формула адресации x86-64:

```
Адрес = База + Индекс × Масштаб + Смещение
```

Где:
- **База** — любой 64-битный регистр (кроме RSP в индексе)
- **Индекс** — любой 64-битный регистр (кроме RSP)
- **Масштаб** — 1, 2, 4 или 8 (умножает индекс)
- **Смещение** — число от -2³¹ до 2³¹-1 (или метка)

Любой из этих компонентов может отсутствовать.

## 3. Пошаговый разбор

### 3.1 Direct addressing — прямая адресация

Это то, что мы уже знаем:

```assembly
section .data
    val dq 42

section .text
    mov rax, [val]       ; rax = 42
```

Адрес берётся прямо из метки. Метка `val` — это число (адрес). Процессор читает память по этому адресу.

Что происходит внутри:
1. Ассемблер подставляет вместо `val` адрес (например, 0x601000)
2. Процессор читает 8 байт по адресу 0x601000
3. Кладёт их в RAX

**Недостаток:** адрес фиксирован. Нельзя изменить его во время работы программы.

---

### 3.2 Register indirect — косвенная адресация через регистр

Теперь самое важное: адрес может лежать **в регистре**.

```assembly
section .data
    val dq 42

section .text
    mov rbx, val          ; rbx = адрес val (например, 0x601000)
    mov rax, [rbx]        ; rax = значение по адресу rbx = 42
```

Здесь:
1. `mov rbx, val` — кладём адрес val в RBX
2. `mov rax, [rbx]` — читаем 8 байт из памяти по адресу, который хранится в RBX

**Это не то же самое, что `mov rax, rbx`**! Сравни:

```assembly
mov rax, rbx    ; КОПИРУЕТ число из RBX в RAX (rax = адрес val)
mov rax, [rbx]  ; ЧИТАЕТ память по адресу из RBX (rax = 42)
```

| Инструкция | RBX | RAX после |
|---|---|---|
| `mov rax, rbx` | 0x601000 | 0x601000 (адрес) |
| `mov rax, [rbx]` | 0x601000 | 42 (значение) |

Квадратные скобки означают: **«зайди по адресу, который в регистре, и возьми то, что там лежит»**.

Зачем это нужно? Теперь мы можем менять адрес во время работы программы:

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rsi, arr          ; rsi = адрес начала массива
    mov eax, [rsi]        ; eax = 10  (arr[0])

    add rsi, 4            ; rsi = адрес arr + 4
    mov eax, [rsi]        ; eax = 20  (arr[1])

    add rsi, 4            ; rsi = адрес arr + 8
    mov eax, [rsi]        ; eax = 30  (arr[2])
```

Мы сдвинули адрес в RSI на 4 байта (размер `dd`) и прочитали следующий элемент.

**Почему именно квадратные скобки?** Потому что `[rbx]` — это то же самое, что `[0x601000]`, только адрес хранится в регистре. Адрес = число в регистре.

---

### 3.3 Base + displacement — базово-смещённая адресация

Часто нужно: **«адрес в регистре + небольшое число»**.

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rbx, arr          ; rbx = адрес arr

    mov eax, [rbx]        ; eax = 10  (arr[0] = rbx + 0)
    mov eax, [rbx + 4]    ; eax = 20  (arr[1] = rbx + 4)
    mov eax, [rbx + 8]    ; eax = 30  (arr[2] = rbx + 8)
    mov eax, [rbx + 12]   ; eax = 40  (arr[3] = rbx + 12)
```

`[rbx + 4]` читается как: **«возьми адрес из RBX, прибавь 4, и прочитай по получившемуся адресу»**.

Это удобнее, чем каждый раз делать `add rbx, 4` — мы не меняем сам регистр, а просто указываем смещение.

Смещение может быть отрицательным:

```assembly
mov eax, [rbx - 4]    ; читаем за 4 байта до адреса в rbx
```

Где это используется?
- **Поля структуры:** у структуры есть база (адрес) и смещения полей
- **Массивы:** если индекс фиксирован
- **Локальные переменные:** в стеке (относительно RBP)

---

### 3.4 Indexed — индексная адресация

А что, если индекс — не число, а **регистр**? То есть мы не знаем заранее, какой элемент нам нужен.

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rsi, 2            ; rsi = индекс (хотим arr[2])
    mov eax, [arr + rsi*4] ; eax = arr[2] = 30
```

Разберём `[arr + rsi*4]`:
1. Берём адрес `arr` (метка, фиксированное число)
2. Берём значение RSI (индекс)
3. Умножаем RSI на 4 (масштаб = размер элемента)
4. Складываем: адрес arr + rsi*4
5. Читаем 4 байта по получившемуся адресу

**Почему умножаем на 4?** Потому что `dd` — 4 байта. Чтобы перейти от индекса к смещению в байтах, нужно умножить индекс на размер элемента.

| Размер элемента | Масштаб |
|---|---|
| db (1 байт) | `*1` |
| dw (2 байта) | `*2` |
| dd (4 байта) | `*4` |
| dq (8 байт) | `*8` |

Процессор поддерживает масштабы: 1, 2, 4, 8 (только эти четыре значения).

Теперь мы можем обращаться к массиву по любому индексу, который лежит в регистре:

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    ; arr[rsi], где rsi = 0, 1, 2, 3, 4
    mov rsi, 0
    mov eax, [arr + rsi*4]    ; eax = 10

    mov rsi, 3
    mov eax, [arr + rsi*4]    ; eax = 40

    mov rsi, 4
    mov eax, [arr + rsi*4]    ; eax = 50
```

Это буквально **arr[rsi]** из языков высокого уровня.

---

### 3.5 Base + index*scale — база + индекс × масштаб

А если и база, и индекс — регистры?

```assembly
section .data
    arr1 dd 10, 20, 30, 40, 50
    arr2 dd 100, 200, 300, 400, 500

section .text
    mov rbx, arr1         ; rbx = адрес arr1
    mov rsi, 2            ; rsi = индекс
    mov eax, [rbx + rsi*4] ; eax = 30  (arr1[2])
```

Здесь `rbx` — база (адрес массива), а `rsi*4` — индекс, умноженный на размер.

Если переключиться на другой массив — достаточно сменить базу:

```assembly
    mov rbx, arr2         ; rbx = адрес arr2
    mov eax, [rbx + rsi*4] ; eax = 300 (arr2[2])
```

Или можно держать индекс в одной руке, а базу — в другой, и менять любую из них.

---

### 3.6 Full addressing — полная форма

Собираем всё вместе:

```
[база + индекс × масштаб + смещение]
```

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rbx, arr           ; rbx = адрес arr
    mov rsi, 1             ; rsi = индекс
    mov eax, [rbx + rsi*4 + 8]  ; eax = arr[1 + 2] = arr[3] = 40
```

Разберём: `rbx + rsi*4 + 8`
1. База: `rbx` = адрес arr
2. Индекс: `rsi` = 1
3. Масштаб: `*4` (размер dd)
4. Смещение: `+8`
5. Итого: arr + 1*4 + 8 = arr + 12 = arr[3]

---

### 3.7 Сводная таблица

| Режим | Пример | Адрес |
|---|---|---|
| Direct | `[val]` | val |
| Register indirect | `[rax]` | RAX |
| Base + disp | `[rax + 10]` | RAX + 10 |
| Base + index | `[rbx + rsi*4]` | RBX + RSI×4 |
| Indexed | `[arr + rsi*4]` | arr + RSI×4 |
| Full | `[rbx + rsi*4 + 10]` | RBX + RSI×4 + 10 |

**Ограничения:**
- База: любой регистр общего назначения (включая RSP, но не RIP)
- Индекс: любой регистр общего назначения **кроме RSP**
- Масштаб: только 1, 2, 4 или 8
- Смещение: 8-, 16- или 32-битное число со знаком

## 4. LEA — Load Effective Address

### 4.1 Зачем нужна LEA?

Есть важное различие между `mov` и `lea`:

```assembly
mov rax, [rbx]    ; ЧИТАЕТ память по адресу rbx
lea rax, [rbx]    ; ВЫЧИСЛЯЕТ адрес (rbx) и кладёт его в rax
```

`lea` (Load Effective Address) **не обращается к памяти**. Она только **вычисляет адрес** и записывает его в регистр.

```assembly
section .data
    val dq 42

section .text
    mov rbx, val          ; rbx = адрес val

    mov rax, [rbx]        ; rax = 42 (прочитали память)
    lea rax, [rbx]        ; rax = адрес val (не читали память)
    mov rax, rbx          ; rax = адрес val (просто копия регистра)
```

**Разница между `mov rax, rbx` и `lea rax, [rbx]`:** в данном случае результат одинаковый. Но LEA умеет вычислять сложные адреса:

```assembly
lea rax, [rbx + rsi*4 + 10]   ; rax = rbx + rsi*4 + 10
```

Эту же операцию через mov так просто не сделаешь:

```assembly
; То же самое через mov (хуже — портит rbx, rsi):
mov rax, rsi
shl rax, 2         ; rax = rsi * 4
add rax, rbx
add rax, 10
```

LEA делает это **одной инструкцией**, быстрее и без изменения регистров.

### 4.2 LEA для арифметики

Поскольку LEA умеет складывать, умножать на 1/2/4/8 и прибавлять смещение, её часто используют для **арифметики без обращения к памяти**:

```assembly
lea rax, [rcx + rdx]      ; rax = rcx + rdx
lea rax, [rcx + rcx*2]    ; rax = rcx * 3
lea rax, [rcx*8]           ; rax = rcx * 8
lea rax, [rcx + rcx*4]    ; rax = rcx * 5
lea rax, [rcx*4 + 5]      ; rax = rcx * 4 + 5
```

**Пример:** умножим RCX на 10 без `mul`:

```assembly
lea rax, [rcx + rcx*4]    ; rax = rcx * 5
lea rax, [rax + rax]      ; rax = (rcx * 5) * 2 = rcx * 10
```

### 4.3 LEA для получения адреса элемента массива

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rsi, 2
    lea rbx, [arr + rsi*4]    ; rbx = адрес arr[2]

    mov eax, [rbx]            ; eax = 30 (читаем по адресу)
```

Это разделяет **вычисление адреса** и **доступ к памяти**.

### 4.4 Чем отличается LEA от MOV?

```assembly
; Случай 1: просто регистр
mov rax, rbx          ; rax = rbx (копия числа)
lea rax, [rbx]        ; rax = rbx (тот же результат)

; Случай 2: чтение памяти
mov rax, [rbx]        ; rax = *rbx (читает 8 байт из памяти)

; Случай 3: вычисление адреса
lea rax, [rbx + 10]   ; rax = rbx + 10 (просто арифметика, без памяти)
mov rax, [rbx + 10]   ; rax = * (rbx + 10) (читает память!)
```

| Инструкция | Обращается к памяти? | Что в RAX? |
|---|---|---|
| `mov rax, rbx` | нет | Значение RBX (число) |
| `lea rax, [rbx]` | нет | Значение RBX (число) |
| `mov rax, [rbx]` | **да** | 8 байт по адресу RBX |
| `lea rax, [rbx+10]` | нет | RBX + 10 |
| `mov rax, [rbx+10]` | **да** | 8 байт по адресу RBX+10 |

### 4.5 Что можно и нельзя в LEA

Внутри скобок LEA работает **та же система адресации**, что и для mov:

```assembly
lea rax, [val]              ; rax = адрес val
lea rax, [rbx]              ; rax = rbx
lea rax, [rbx + 10]         ; rax = rbx + 10
lea rax, [arr + rsi*4]      ; rax = arr + rsi*4
lea rax, [rbx + rsi*4 + 10] ; rax = rbx + rsi*4 + 10
```

Но обрати внимание: LEA принимает **адресный операнд**, а не произвольное выражение. Нельзя написать:

```assembly
lea rax, rbx + rcx        ; НЕЛЬЗЯ! Складывать регистры без скобок
lea rax, [rbx + rcx]      ; МОЖНО (оба — регистры)
```

## 5. Полная программа

```assembly
; addressing.asm — демонстрация всех режимов адресации
section .data
    ; Массив чисел (dd = 4 байта каждое)
    arr dd 10, 20, 30, 40, 50
    arr_len equ 5

    ; Строка для вывода
    msg_arr   db "Array: ", 0
    msg_lea   db "LEA addr: ", 0
    msg_el    db " element: ", 0
    msg_full  db "Full mode: ", 0
    newline   db 10

section .text
    global _start

_start:
    ; ──────────────────────────────────────────────────
    ; 1. Direct addressing — arr[2] напрямую
    ; ──────────────────────────────────────────────────
    mov eax, [arr + 8]      ; arr + 2*4 = arr + 8 -> 30
    ; eax = 30

    ; ──────────────────────────────────────────────────
    ; 2. Register indirect — адрес в регистре
    ; ──────────────────────────────────────────────────
    mov rbx, arr            ; rbx = адрес arr
    mov eax, [rbx]          ; читаем по адресу rbx -> 10 (arr[0])

    ; ──────────────────────────────────────────────────
    ; 3. Base + displacement — регистр + смещение
    ; ──────────────────────────────────────────────────
    mov ebx, [rbx + 12]     ; arr + 12 = arr[3] = 40
    ; ebx = 40

    ; ──────────────────────────────────────────────────
    ; 4. Indexed — метка + регистр*масштаб
    ; ──────────────────────────────────────────────────
    mov rsi, 4              ; индекс = 4
    mov eax, [arr + rsi*4]  ; arr[4] = 50
    ; eax = 50

    ; ──────────────────────────────────────────────────
    ; 5. Full — база + индекс*масштаб + смещение
    ; ──────────────────────────────────────────────────
    mov rbx, arr            ; база = arr
    mov rsi, 1              ; индекс = 1
    mov eax, [rbx + rsi*4 + 8]  ; arr + 1*4 + 8 = arr + 12 = arr[3] = 40
    ; eax = 40

    ; ──────────────────────────────────────────────────
    ; 6. LEA — вычисляем адрес, не читая память
    ; ──────────────────────────────────────────────────
    mov rsi, 2
    lea rdx, [arr + rsi*4]  ; rdx = адрес arr[2] (вычисление, без доступа к памяти)
    ; rdx = arr + 2*4 = arr + 8

    ; ──────────────────────────────────────────────────
    ; 7. Проход по массиву (паттерн цикла)
    ; ──────────────────────────────────────────────────
    ; Пройдём по arr и запишем в каждый элемент его индекс*10
    ; arr[0] = 0, arr[1] = 10, arr[2] = 20, arr[3] = 30, arr[4] = 40

    mov rbx, arr            ; rbx = адрес массива
    mov rsi, 0              ; rsi = индекс (начинаем с 0)
    mov rcx, arr_len        ; rcx = количество элементов

.loop:
    ; вычисляем: значение = индекс * 10
    mov rax, rsi
    mov r8, 10
    mul r8                  ; rax = rsi * 10

    ; записываем в arr[rsi]
    mov [rbx + rsi*4], eax  ; arr[rsi] = rsi * 10

    inc rsi                 ; индекс++
    dec rcx                 ; счётчик--
    cmp rcx, 0
    jne .loop               ; если не ноль — повторяем

    ; Теперь arr = {0, 10, 20, 30, 40}

    ; Выведем что-нибудь для проверки
    mov rsi, 3
    mov eax, [arr + rsi*4]  ; arr[3] = 30
    ; eax = 30

    ; ──────────────────────────────────────────────────
    ; Выход с кодом = arr[3] = 30
    ; ──────────────────────────────────────────────────
    mov rax, 60
    mov rdi, 30
    syscall
```

Собери и запусти:

```bash
nasm -f elf64 addressing.asm -o addressing.o
ld addressing.o -o addressing
./addressing
echo $?    # должно вывести 30
```

## 6. Частые ошибки

### Ошибка 1: путать `mov rax, [rbx]`, `mov rax, rbx` и `lea rax, [rbx]`

```assembly
mov rax, [rbx]    ; читает 8 байт из памяти по адресу rbx
mov rax, rbx      ; копирует сам адрес (число) из rbx в rax
lea rax, [rbx]    ; вычисляет адрес rbx (просто копия, без памяти)
```

Вопрос: что будет в RAX после каждого?
- Если rbx = 0x601000, а по адресу 0x601000 лежит число 42:
  - `mov rax, [rbx]` → rax = 42
  - `mov rax, rbx` → rax = 0x601000
  - `lea rax, [rbx]` → rax = 0x601000

### Ошибка 2: забыть про масштаб (умножение на размер элемента)

```assembly
section .data
    arr dd 10, 20, 30, 40, 50

section .text
    mov rsi, 2
    mov eax, [arr + rsi]   ; НЕПРАВИЛЬНО! rsi = 2, читаем arr + 2 (не arr + 8)
    mov eax, [arr + rsi*4] ; ПРАВИЛЬНО! arr + 2*4 = arr + 8 = arr[2]
```

Для `dd` (4 байта) нужен масштаб `*4`. Для `dq` — `*8`. Для `db` — `*1` (можно не писать).

### Ошибка 3: использовать RSP как индекс

```assembly
mov rax, [rsp + rsp*4]   ; ОШИБКА! RSP нельзя использовать как индекс
mov rax, [rbx + rsp]     ; OK: RSP может быть базой, но не индексом
```

### Ошибка 4: использовать два регистра без масштаба

```assembly
mov rax, [rbx + rcx]     ; OK (масштаб по умолчанию 1)
lea rax, [rbx + rcx]     ; OK (сумма двух регистров)
; Но нельзя умножать регистр на произвольное число:
mov rax, [rbx + rsi*3]   ; ОШИБКА! Масштаб только 1, 2, 4, 8
```

### Ошибка 5: LEA с обращением к памяти

```assembly
lea rax, [rbx]    ; OK — только вычисление адреса
lea rax, [val]    ; OK — rax = адрес val (не читает val!)
mov rax, [val]    ; читает val
```

LEA никогда не обращается к памяти. Она только вычисляет адрес и кладёт его в регистр.

### Ошибка 6: путать `mov [rbx + rsi*4], eax` и `mov eax, [rbx + rsi*4]`

```assembly
mov [rbx + rsi*4], eax    ; ЗАПИСЬ: eax → память по адресу rbx + rsi*4
mov eax, [rbx + rsi*4]    ; ЧТЕНИЕ: память по адресу rbx + rsi*4 → eax
```

Запись — регистр слева, адрес в скобках справа.
Чтение — адрес в скобках слева, регистр справа.

## 7. Упражнения

1. **Register indirect.** Объяви `val dq 99`. Положи адрес val в RBX. Прочитай значение через `[rbx]`. Выйди с ним.

2. **Base + displacement.** Объяви массив `arr dd 5, 10, 15, 20, 25`. Прочитай arr[4] (индекс 4) через `[адрес + смещение]` без умножения. Выйди с кодом 25.

3. **Indexed.** Тот же массив. Положи индекс 2 в RSI. Прочитай `[arr + rsi*4]`. Выйди с 15.

4. **Полный режим.** Объяви `arr dq 100, 200, 300, 400, 500`. Используя RBX как базу (адрес arr), RSI = 1, смещение = 16, прочитай элемент. Какой индекс получился? Должно быть 300.

5. **LEA как арифметика.** Вычисли `rax = rcx * 4 + rdx + 10` через одну LEA (без add/mul). Проверь: положи rcx = 7, rdx = 3. Результат = 7*4 + 3 + 10 = 41.

6. **Проход по массиву.** Объяви массив `arr dd 1, 2, 3, 4, 5`. Пройди по нему в цикле (как в примере) и замени каждый элемент на его квадрат. Выйди с кодом = arr[4] (должно быть 25).

7. **Два массива.** Объяви `a dd 1, 2, 3` и `b dd 10, 20, 30`. Пройди по индексам, сложи a[i] + b[i] и запиши в c[i] (c тоже dd, 3 элемента). Выйди с c[1] (должно быть 22).

8. **LEA против MOV.** Напиши два варианта кода, которые делают `rax = rbx + rcx*8 + 15`: один через LEA, второй через mov/add/shift. Сравни длину (посмотри через `objdump -d`).
