# [02] Регистры и mov

> **Трек:** Assembly · **Уровень:** 02 · **Сложность:** ★☆☆☆☆

## 1. Какую проблему решаем?

В прошлом уроке мы написали программу, которая выводит "Hello!". Но строка была жёстко задана в коде. Что, если мы хотим вывести **число**, которое получилось в результате вычисления?

Чтобы манипулировать данными, нужно понимать **регистры** — ячейки внутри процессора, где живут данные во время выполнения программы. И инструкцию **mov** — главный способ перемещать данные.

## 2. Все регистры 64-битного процессора

У процессора (в режиме 64 бит) есть такие регистры общего назначения:

```
64-битный     32-битный     16-битный   8-бит (старший)  8-бит (младший)
──────────    ──────────    ──────────   ──────────────   ──────────────
RAX           EAX           AX           AH                AL
RBX           EBX           BX           BH                BL
RCX           ECX           CX           CH                CL
RDX           EDX           DX           DH                DL
RSI           ESI           SI           ─                 SIL
RDI           EDI           DI           ─                 DIL
RBP           EBP           BP           ─                 BPL
RSP           ESP           SP           ─                 SPL

R8            R8D           R8W           ─                R8B
R9            R9D           R9W           ─                R9B
R10           R10D          R10W          ─                R10B
...
R15           R15D          R15W          ─                R15B
```

### Почему у регистра есть части?

Регистр RAX — это 64 бита (8 байт). Но часто нужно работать только с частью:
- Если ты пишешь в **EAX** — ты меняешь только младшие 32 бита RAX. Старшие 32 бита **обнуляются** (важно!)
- Если ты пишешь в **AX** — ты меняешь только младшие 16 бит. Старшие биты не трогаются
- Если ты пишешь в **AL** — ты меняешь только младшие 8 бит
- Если ты пишешь в **AH** — ты меняешь биты 8-15 (второй байт)

```
RAX (64 бита)
┌──────────────────────────────────────────────────────────────┐
│                        RAX                                   │
│  ┌──────────────────────────┬───────────────────────────────┐│
│  │        EAX               │                               ││
│  │  ┌──────────┬───────────┐│                               ││
│  │  │   AX     │           ││                               ││
│  │  │ ┌──┬────┐│           ││                               ││
│  │  │ │AH│ AL ││           ││                               ││
│  │  │ └──┴────┘│           ││                               ││
│  │  └──────────┴───────────┘│                               ││
│  └──────────────────────────┴───────────────────────────────┘│
└──────────────────────────────────────────────────────────────┘
  63                       31  23 15 7                       0
```

### Регистры специального назначения

| Регистр | Назначение |
|---------|-----------|
| **RIP** | Instruction Pointer — адрес текущей инструкции. Нельзя писать напрямую |
| **RSP** | Stack Pointer — вершина стека (об этом в уроке 08) |
| **RBP** | Base Pointer — база стекового фрейма (для функций) |
| **RFLAGS** | Флаги: ZF, CF, SF, OF и другие |

RSP, RBP и RFLAGS — тоже регистры, но их не используют для хранения обычных данных. RSP и RBP нужны для стека, RFLAGS меняется автоматически после инструкций.

## 3. Инструкция mov

`mov` — самая используемая инструкция в ассемблере. Она **копирует значение** из источника в назначение.

**Синтаксис:** `mov назначение, источник`

```
mov rax, 5        ; rax = 5           (непосредственное значение)
mov rbx, rax      ; rbx = rax         (регистр → регистр)
mov rcx, [addr]   ; rcx = *addr       (память → регистр)
mov [addr], rdx   ; *addr = rdx       (регистр → память)
```

**Важно:** mov не умеет копировать из памяти в память напрямую:
```assembly
mov [addr1], [addr2]   ; ОШИБКА! так нельзя
```
Нужно через регистр:
```assembly
mov rax, [addr2]       ; прочитали из addr2 в rax
mov [addr1], rax        ; записали из rax в addr1
```

### 3.1 mov с непосредственным значением (immediate)

```assembly
mov rax, 42        ; rax = 42
mov rbx, 0xFF      ; rbx = 255
mov rcx, 0x1000    ; rcx = 4096
mov al, 10         ; al = 10 (младший байт RAX)
mov ah, 20         ; ah = 20 (старший байт AX)
```

Число записывается прямо в инструкцию. Можно в десятичной (42) или hex (0xFF).

### 3.2 mov регистр → регистр

```assembly
mov rax, rbx       ; скопировать RBX в RAX
mov rsi, rdi       ; скопировать RDI в RSI
mov r8, r9         ; скопировать R9 в R8
```

После `mov rax, rbx` в обоих регистрах одно и то же число. Исходный (rbx) не меняется — это **копирование**, а не перемещение.

### 3.3 mov между частями регистров

```assembly
mov rax, 0x123456789ABCDEF0   ; rax = большое число
mov ebx, eax                  ; ebx = младшие 32 бита RAX
                                ; старшие 32 бита RBX = 0!
mov cx, ax                    ; cx = младшие 16 бит AX
mov dl, al                    ; dl = младшие 8 бит AL
```

**Важное правило:** когда ты пишешь в 32-битную часть (EAX, EBX...), старшие 32 бита регистра **обнуляются**. Когда пишешь в 16-битную (AX) или 8-битную (AL, AH) — остальные биты не меняются.

Пример:
```assembly
mov rax, 0xFFFFFFFFFFFFFFFF    ; RAX = все единицы
mov eax, 0                    ; RAX = 0x0000000000000000 (старшие 32 бита обнулились!)

mov rax, 0xFFFFFFFFFFFFFFFF    ; RAX = все единицы
mov ax, 0                     ; RAX = 0xFFFFFFFFFFFF0000 (только младшие 16 бит = 0)
```

### 3.4 mov с памятью (квадратные скобки)

Квадратные скобки `[]` означают: **«не адрес, а то, что лежит по этому адресу»**.

```assembly
section .data
    my_var dq 42          ; выделить 8 байт, положить число 42

section .text
    global _start
_start:
    mov rax, my_var       ; rax = АДРЕС переменной my_var (число)
    mov rbx, [my_var]     ; rbx = ЗНАЧЕНИЕ по адресу my_var (42)
```

Разница:
```
mov rax, my_var    → в RAX попадёт адрес my_var (например, 0x601000)
mov rbx, [my_var]  → в RBX попадёт то, что лежит по адресу my_var (42)
```

### 3.5 Правила типов для mov с памятью

Важно указывать **размер** через размер регистра:

```assembly
section .data
    b db 10        ; 1 байт
    w dw 1000      ; 2 байта
    d dd 100000    ; 4 байта
    q dq 123456    ; 8 байт

section .text
    mov al, [b]    ; читаем 1 байт → AL
    mov ax, [w]    ; читаем 2 байта → AX
    mov eax, [d]   ; читаем 4 байта → EAX
    mov rax, [q]   ; читаем 8 байт → RAX
```

Нельзя прочитать 8 байт в AL (AL — 1 байт). Нельзя прочитать 1 байт в RAX и надеяться, что остальные 7 байт обнулятся (они будут мусором, если не обнулить RAX перед этим).

## 4. Программа: вывести значение регистра

Теперь мы умеем работать с регистрами. Но как увидеть их значение? Только через write.

Напишем программу, которая:
1. Кладёт числа в регистры
2. Выводит результат (пока — через строку в .data, потому что число в регистре — это число, а не строка)

```assembly
; registers.asm — демонстрация mov и регистров
section .data
    msg db "RAX = ", 0     ; 0 = null terminator (необязателен для write)
    newline db 10

section .text
    global _start

_start:
    ; mov imm → reg
    mov rax, 42
    mov rbx, 100
    mov rcx, 0xFF

    ; mov reg → reg
    mov rdx, rax        ; rdx = 42

    ; mov с частями регистров
    mov rax, 0xABCD
    mov eax, 0          ; RAX = 0 (старшие 32 бита обнулились)
    mov ax, 42          ; RAX = 42 (только младшие 16 бит)

    ; выведем что-нибудь (пока просто строку)
    mov rax, 1          ; write
    mov rdi, 1          ; stdout
    mov rsi, msg        ; "RAX = "
    mov rdx, 6          ; 6 байт
    syscall

    ; newline
    mov rax, 1
    mov rdi, 1
    mov rsi, newline
    mov rdx, 1
    syscall

    ; exit(0)
    mov rax, 60
    xor rdi, rdi
    syscall
```

Вывод:
```
RAX = 
```

## 5. Какой размер у инструкции mov?

Разные формы mov занимают разное количество байт в памяти:

```assembly
mov rax, 42         ; 7 байт: 48 C7 C0 2A 00 00 00
mov eax, 42         ; 5 байт: B8 2A 00 00 00
mov al, 42          ; 2 байта: B0 2A
mov rax, rbx        ; 3 байта: 48 89 D8
```

Не нужно запоминать эти коды. Просто знай: инструкции с 64-битными регистрами занимают больше места. Если число помещается в 32 бита — используй `eax` вместо `rax`.

## 6. Что можно и что нельзя делать с mov?

```
mov reg, imm         ✅  rax = 5
mov reg, reg         ✅  rbx = rax
mov reg, [mem]       ✅  rcx = [addr]
mov [mem], reg       ✅  [addr] = rdx
mov [mem], imm       ✅  [addr] = 42
mov [mem], [mem]     ❌  так нельзя, через регистр
mov reg, [mem+reg]   ✅  сложная адресация (следующий урок)
```

## 7. Частые ошибки

### Ошибка 1: путать адрес и значение
```assembly
mov rax, my_var      ; адрес (число)
mov rbx, [my_var]    ; значение
```
Если нужно значение — ставь квадратные скобки.

### Ошибка 2: несоответствие размеров
```assembly
mov [my_byte], rax   ; запись 8 байт в 1-байтовую переменную — порча памяти!
mov rax, [my_byte]   ; чтение 8 байт из 1-байтовой — мусор в остальных!
```
Следи за размером. В AL — 1 байт, в AX — 2, в EAX — 4, в RAX — 8.

### Ошибка 3: забыть про обнуление старших 32 бит
```assembly
mov rax, 0xFFFFFFFFFFFFFFFF
mov eax, 5           ; RAX = 5, не 0xFFFFFFFF00000005!
```

### Ошибка 4: mov [mem], [mem]
```assembly
mov [x], [y]         ; синтаксическая ошибка!
```
Через регистр:
```assembly
mov rax, [y]
mov [x], rax
```
