# [03] Арифметика

> **Трек:** Assembly · **Уровень:** 03 · **Сложность:** ★★☆☆☆

## 1. Какую проблему решаем?

Компьютеры были изобретены для счёта. Складывать, вычитать, умножать и делить числа — базовое умение любого процессора.

В этом уроке ты научишься:
- Складывать и вычитать (`add`, `sub`, `inc`, `dec`)
- Умножать (`mul`, `imul`)
- Делить (`div`, `idiv`)
- Понимать флаги (ZF, CF, SF, OF) — они пригодятся в следующем уроке для переходов

## 2. Сложение и вычитание

### add — сложение

```assembly
add rax, 5        ; rax = rax + 5
add rbx, rcx      ; rbx = rbx + rcx
add rax, [var]    ; rax = rax + значение из памяти
```

Пример:
```assembly
mov rax, 10
add rax, 20        ; rax = 10 + 20 = 30
```

После `add` меняются флаги:
- **ZF** (Zero Flag) = 1, если результат 0
- **CF** (Carry Flag) = 1, если был перенос из старшего бита (результат не влез в 64 бита)
- **SF** (Sign Flag) = 1, если результат отрицательный (старший бит = 1)
- **OF** (Overflow Flag) = 1, если знаковое переполнение

### sub — вычитание

```assembly
sub rax, 5        ; rax = rax - 5
sub rbx, rcx      ; rbx = rbx - rcx
sub rax, [var]    ; rax = rax - значение из памяти
```

Пример:
```assembly
mov rax, 100
sub rax, 30        ; rax = 100 - 30 = 70
```

### inc и dec — инкремент и декремент

```assembly
inc rax           ; rax = rax + 1 (увеличить на 1)
dec rbx           ; rbx = rbx - 1 (уменьшить на 1)
```

`inc` и `dec` — это просто короткая запись для `add x, 1` и `sub x, 1`. Они занимают меньше места в памяти.

Пример:
```assembly
mov rax, 0
inc rax            ; rax = 1
inc rax            ; rax = 2
dec rax            ; rax = 1
```

## 3. Флаги подробнее

После `add` или `sub` процессор обновляет флаги. Это работает автоматически.

### ZF — Zero Flag
```assembly
mov rax, 10
sub rax, 10        ; rax = 0 → ZF = 1
```

### CF — Carry Flag (для беззнаковых чисел)
```assembly
mov rax, 0xFFFFFFFFFFFFFFFF   ; максимальное 64-битное число
add rax, 1                    ; rax = 0, CF = 1 (перенос из старшего бита)
```
CF = 1 означает, что результат не поместился в 64 бита.

### SF — Sign Flag (для знаковых чисел)
```assembly
mov rax, 5
sub rax, 10        ; rax = -5 → SF = 1 (старший бит = 1)
```

### OF — Overflow Flag (знаковое переполнение)
```assembly
mov rax, 0x7FFFFFFFFFFFFFFF   ; максимальное положительное 64-битное число
add rax, 1                    ; rax = 0x8000000000000000 (отрицательное!), OF = 1
```

## 4. Программа: считаем на регистрах

```assembly
; arithmetic.asm — демонстрация add/sub/inc/dec
section .data
    msg1 db "10 + 20 = 30", 10
    len1 equ $ - msg1
    msg2 db "100 - 30 = 70", 10
    len2 equ $ - msg2

section .text
    global _start

_start:
    ; === сложение ===
    mov rax, 10
    add rax, 20         ; rax = 30

    ; выведем (пока строку, числа потом научимся)
    mov rax, 1
    mov rdi, 1
    mov rsi, msg1
    mov rdx, len1
    syscall

    ; === вычитание ===
    mov rax, 100
    sub rax, 30         ; rax = 70

    mov rax, 1
    mov rdi, 1
    mov rsi, msg2
    mov rdx, len2
    syscall

    ; === inc/dec ===
    mov rbx, 0
    inc rbx             ; rbx = 1
    inc rbx             ; rbx = 2
    dec rbx             ; rbx = 1

    ; exit(0)
    mov rax, 60
    xor rdi, rdi
    syscall
```

## 5. Умножение

### mul — беззнаковое умножение

`mul` умножает **беззнаковые** числа. У неё только один операнд — множитель. Множимое всегда в RAX.

```
mul rbx   →  RAX = RAX * RBX
            RDX = старшие 64 бита результата (если не влезает в 64 бита)
```

Пример:
```assembly
mov rax, 5
mov rbx, 3
mul rbx            ; RAX = 5 * 3 = 15, RDX = 0
```

Если результат не влезает в 64 бита, старшая часть попадает в RDX:
```assembly
mov rax, 0xFFFFFFFFFFFFFFFF   ; это 2^64 - 1
mov rbx, 2
mul rbx            ; RAX = 0xFFFFFFFFFFFFFFFE, RDX = 1
                   ; полный результат: (RDX << 64) | RAX = 2^65 - 2
```

Для умножения на маленькие числа (до 2^64) можно игнорировать RDX — он будет 0.

### imul — знаковое умножение

`imul` умножает **знаковые** числа. У неё есть несколько форм:

```assembly
imul rbx           ; RAX = RAX * RBX (как mul, но знаковое)
imul rax, rbx      ; RAX = RAX * RBX
imul rax, rbx, 5   ; RAX = RBX * 5
```

Пример:
```assembly
mov rax, -5
mov rbx, 3
imul rbx           ; RAX = -15 (0xFFFFFFFFFFFFFFF1 в hex, т.к. отрицательное)
```

## 6. Деление

### div — беззнаковое деление

`div` делит **беззнаковые** числа. У неё один операнд — делитель. Делимое — всегда RDX:RAX (128 бит).

```
div rbx   →   RAX = (RDX:RAX) / RBX   (частное)
              RDX = (RDX:RAX) % RBX   (остаток)
```

Обычно RDX = 0 перед делением (делим 64-битное число):
```assembly
mov rax, 15        ; делимое
xor rdx, rdx       ; RDX = 0 (старшая часть)
mov rbx, 4         ; делитель
div rbx            ; RAX = 15 / 4 = 3 (частное)
                   ; RDX = 15 % 4 = 3 (остаток)
```

Пример с остатком:
```assembly
mov rax, 7
xor rdx, rdx
mov rbx, 3
div rbx            ; RAX = 2, RDX = 1  (7 / 3 = 2, остаток 1)
```

**ВАЖНО:** Если делитель меньше или равен 0, или частное не влезает в 64 бита — программа упадёт с `SIGFPE` (деление на ноль). Всегда проверяй делитель!

### idiv — знаковое деление

То же, что и `div`, но для знаковых чисел. Перед делением нужно расширить знак делимого:

```assembly
mov rax, -15       ; делимое (отрицательное)
cqo                ; расширить знак RAX в RDX
                   ; если RAX >= 0: RDX = 0
                   ; если RAX < 0:  RDX = -1 (0xFFFFFFFFFFFFFFFF)
mov rbx, 4         ; делитель
idiv rbx           ; RAX = -3 (частное), RDX = -3 (остаток)
```

`cqo` (Convert Quadword to Octaword) — расширяет знак RAX в RDX. Нужна для знакового деления, чтобы RDX был правильным (0 для положительных, -1 для отрицательных).

## 7. Полная программа: калькулятор

```assembly
; calc.asm — вычисляет (5 + 3) * 2 - 4 / 2
section .data
    expr db "((5 + 3) * 2) - (4 / 2) = ??", 10
    len_expr equ $ - expr

section .text
    global _start

_start:
    ; === вычисляем ===
    mov rax, 5
    add rax, 3          ; rax = 8
    mov rbx, 2
    mul rbx             ; rax = 8 * 2 = 16 (rbx = 2)

    ; сохраним результат
    mov rcx, rax        ; rcx = 16

    ; вычислим 4 / 2
    mov rax, 4
    xor rdx, rdx
    mov rbx, 2
    div rbx             ; rax = 2

    ; вычтем: rcx - rax
    sub rcx, rax        ; rcx = 16 - 2 = 14

    ; выведем строку
    mov rax, 1
    mov rdi, 1
    mov rsi, expr
    mov rdx, len_expr
    syscall

    ; exit(14) — чтобы проверить результат через echo $?
    mov rax, 60
    mov rdi, 14         ; код возврата = результат
    syscall
```

После запуска: `echo $?` должен показать 14.

## 8. Арифметика с памятью

Можно складывать/вычитать прямо из памяти:

```assembly
section .data
    a dq 10
    b dq 20
    result dq 0

section .text
    mov rax, [a]
    add rax, [b]        ; rax = a + b
    mov [result], rax    ; result = a + b
```

Но нельзя: `add [a], [b]` — обе памяти запрещено. Нужно через регистр.

## 9. Частые ошибки

### Ошибка 1: забыть обнулить RDX перед div
```assembly
mov rax, 15
mov rbx, 4
div rbx            ; RDX может быть любым! Результат будет неверным.
```
**Решение:** всегда ставь `xor rdx, rdx` перед `div` (или `cqo` перед `idiv`).

### Ошибка 2: деление на ноль
```assembly
mov rax, 5
xor rdx, rdx
mov rbx, 0
div rbx            ; SIGFPE! Программа упадёт.
```

### Ошибка 3: перепутать mul и imul
```assembly
mov rax, -5        ; если считать как беззнаковое: это 0xFFFFFFFFFFFFFFFB
mov rbx, 3
mul rbx            ; rax = 0xFFFFFFFFFFFFFFF1  (не -15 в беззнаковом!)
imul rbx           ; rax = -15 (правильно)
```
Если числа могут быть отрицательными — используй `imul`. Если только положительные — `mul`.

### Ошибка 4: забыть про сохранность регистров
После вызова syscall (write, read) RAX меняется — в нём возвращается количество байт. Если ты сохранял в RAX промежуточный результат — он пропал.
