# [04] Память: секция .data

> **Трек:** Assembly · **Уровень:** 04 · **Сложность:** ★★☆☆☆

## 1. Какую проблему решаем?

До сих пор мы работали только с регистрами. Регистров всего 16, каждый хранит одно число (8 байт). Для серьёзных программ этого мало.

Где хранить:
- Строки (десятки байт)?
- Массивы (сотни чисел)?
- Структуры (набор связанных данных)?

Для этого есть **секция .data** — область памяти, где можно объявить переменные любого размера.

## 2. Что такое секция .data?

Когда ты пишешь:

```assembly
section .data
    msg db "Hello!", 10
```

ты говоришь ассемблеру:
1. Выдели место в памяти (в исполняемом файле)
2. Заполни это место байтами: 72 101 108 108 111 33 10
3. Запомни адрес начала как `msg`

Когда программа загружается в память, секция .data попадает в RAM по адресу `msg`. И вся программа может обращаться к этим байтам по адресу `msg`.

## 3. Директивы выделения памяти

### db — Define Byte (1 байт)

```assembly
; Каждый элемент = 1 байт
byte_var db 42                    ; один байт со значением 42
byte_arr db 10, 20, 30, 40       ; 4 байта: 0A 14 1E 28
char_arr db "Hello", 0           ; 6 байт: 48 65 6C 6C 6F 00
mix     db 1, "AB", 2            ; 4 байта: 01 41 42 02
```

**db = define byte**. Каждый элемент занимает 1 байт. Можно смешивать числа, строки, символы.

### dw — Define Word (2 байта)

```assembly
word_var dw 1000                  ; 2 байта: E8 03 (little-endian: 0x03E8 = 1000)
word_arr dw 1, 2, 3               ; 6 байт: 01 00 02 00 03 00
```

**dw = define word.** Каждый элемент = 2 байта (16 бит). Маленькое число!

### dd — Define Double word (4 байта)

```assembly
dword_var dd 100000               ; 4 байта: A0 86 01 00 (0x186A0 = 100000)
dword_arr dd 10, 20               ; 8 байт: 0A 00 00 00 14 00 00 00
```

**dd = define double word.** Каждый элемент = 4 байта (32 бита). Числа до ~4 миллиардов.

### dq — Define Quad word (8 байт)

```assembly
qword_var dq 123456789            ; 8 байт
qword_arr dq 1, 2                 ; 16 байт
```

**dq = define quad word.** Каждый элемент = 8 байт (64 бита). Огромные числа.

### resb, resw, resd, resq — резервирование без инициализации

```assembly
buffer resb 256                   ; 256 байт, не заполнены (будут нули)
array  resd 100                   ; 100 * 4 = 400 байт
bigbuf resq 1024                  ; 1024 * 8 = 8192 байт
```

`resb` = reserve bytes. Выделяет место, но не заполняет значениями. Используется для буферов.

## 4. Как это выглядит в памяти?

```assembly
section .data
    a db 10           ; a → [10]
    b dw 1000         ; b → [232, 3]   (0x03E8 = 1000, little-endian)
    c dd 100000       ; c → [160, 134, 1, 0]  (0x000186A0 = 100000)
    d dq 123456       ; d → [64, 226, 1, 0, 0, 0, 0, 0]
```

В памяти (little-endian — младший байт по младшему адресу):
```
Адрес:  0x00  0x01  0x02  0x03  0x04  0x05  0x06  0x07  0x08  0x09
       [ 0A ] [ E8 ] [ 03 ] [ A0 ] [ 86 ] [ 01 ] [ 00 ] [ 40 ] [ E2 ] ...
        ↑a     ↑b           ↑c                 ↑d
```

### Что такое little-endian?

Число 1000 в hex = 0x03E8. В памяти (little-endian): сначала младший байт (0xE8), потом старший (0x03).

Если бы было big-endian: 0x03, 0xE8. Но x86 использует little-endian.

**Почему это важно?** Когда ты читаешь байты из памяти в RAX, порядок байт — little-endian. Если ты сохранил строку в .data и читаешь её побайтово — неважно. Если сохранил число — имей в виду.

## 5. Метки: имена для адресов

```assembly
section .data
    msg   db "Result: ", 0
    len   equ $ - msg
    count dq 0
```

**msg** — метка. Ассемблер подставляет адрес первого байта строки. Где бы ты ни написал `msg`, это будет адрес (например, 0x601000).

**count** — ещё одна метка. Адрес переменной `count`.

Имена меток:
- Могут содержать буквы, цифры, `_`
- Не могут начинаться с цифры
- Регистрозависимы (`Msg` ≠ `msg`)
- Лучше давать осмысленные имена

## 6. Как читать из .data

```assembly
section .data
    my_num dq 42

section .text
    global _start
_start:
    ; Читаем значение
    mov rax, [my_num]   ; rax = 42

    ; Читаем адрес
    mov rbx, my_num     ; rbx = адрес my_num (число, а не значение)

    ; Читаем 1 байт
    mov al, [my_num]    ; al = 42 (но только 1 байт!)

    ; Записываем
    mov qword [my_num], 100  ; my_num = 100
```

**Разница между `my_num` и `[my_num]`:**
- `my_num` — адрес (например, 0x601000)
- `[my_num]` — значение по этому адресу (42)

## 7. Массивы в .data

```assembly
section .data
    ; Массив из 5 чисел (каждое по 4 байта)
    arr dd 10, 20, 30, 40, 50
    ; arr[0] = 10, arr[1] = 20, arr[2] = 30, arr[3] = 40, arr[4] = 50

    ; Массив байт (строка)
    str db "Hello", 0
    ; str[0] = 'H', str[1] = 'e', str[2] = 'l', str[3] = 'l', str[4] = 'o', str[5] = 0
```

Как обратиться к элементу массива?
```assembly
; arr[2] (третий элемент, индекс 2)
; arr dd 10, 20, 30, 40, 50
; каждый элемент = 4 байта, поэтому arr[2] = arr + 2*4 = arr + 8

mov rax, [arr]        ; читаем первые 8 байт = arr[0] и arr[1] (плохо!)
mov eax, [arr]        ; читаем первые 4 байта = arr[0] = 10 (хорошо)
mov eax, [arr + 4]    ; читаем байты 4-7 = arr[1] = 20
mov eax, [arr + 8]    ; читаем байты 8-11 = arr[2] = 30
```

Продвинутая адресация (следующий урок):
```assembly
mov rsi, 8
mov eax, [arr + rsi]  ; читаем arr[2] через регистр
```

## 8. equ и $

`equ` (equate) — не выделяет память, а вычисляет значение прямо во время ассемблирования.

```assembly
section .data
    msg db "Hello, World!", 10
    len equ $ - msg
```

Что здесь происходит:
1. `msg db "Hello, World!", 10` — выделяет 14 байт в памяти. Адрес первого байта запоминается как `msg`.
2. `$` — текущий адрес в этой секции. После строки это адрес первого байта после `msg`.
3. `$ - msg` — разница адресов = длина строки в байтах.

Если строка длиннее или короче — `len` пересчитается автоматически. Это лучше, чем писать число вручную.

Можно использовать equ для констант:
```assembly
section .data
    ; константы, не занимают память
    SYS_EXIT  equ 60
    SYS_WRITE equ 1
    STDOUT    equ 1

section .text
    mov rax, SYS_WRITE    ; rax = 1
    mov rdi, STDOUT       ; rdi = 1
```

## 9. Программа: работа с .data

```assembly
; data.asm — демонстрация .data
section .data
    msg db "Array element: ", 0
    arr dd 10, 20, 30, 40, 50
    newline db 10

section .text
    global _start

_start:
    ; читаем arr[2] = 30
    mov eax, [arr + 8]    ; arr + 8 байт = arr[2]
    ; eax = 30

    ; перезаписываем arr[2]
    mov dword [arr + 8], 99   ; arr[2] = 99

    ; выводим строку
    mov rax, 1
    mov rdi, 1
    mov rsi, msg
    mov rdx, 16
    syscall

    ; exit(0)
    mov rax, 60
    xor rdi, rdi
    syscall
```

`mov dword [arr + 8], 99` — запись 4 байт (dword) со значением 99 по адресу arr+8. `dword` перед `[arr+8]` — это указатель размера, потому что ассемблер не знает, сколько байт писать.

## 10. Таблица: когда какой размер использовать

| Ситуация | Размер | Директива | Регистр |
|----------|--------|-----------|---------|
| Символ (буква) | 1 байт | db | AL |
| Флаг (0/1) | 1 байт | db | AL |
| Маленькое число (до 255) | 1 байт | db | AL |
| Число до 65535 | 2 байта | dw | AX |
| Число до 4 млрд | 4 байта | dd | EAX |
| Большое число | 8 байт | dq | RAX |
| Адрес (указатель) | 8 байт | dq | RAX |
| Буфер, строка | 1 байт × N | db + resb | AL + счётчик |

## 11. Частые ошибки

### Ошибка 1: забыть размер при записи в память
```assembly
mov [var], 100              ; ОШИБКА: какого размера 100?
mov dword [var], 100        ; OK: пишем 4 байта
mov qword [var], 100        ; OK: пишем 8 байт
```

### Ошибка 2: читать больше, чем объявлено
```assembly
section .data
    b db 10         ; всего 1 байт

section .text
    mov rax, [b]    ; читаем 8 байт! 7 из них — мусор!
```
Читай ровно столько, сколько объявил: `mov al, [b]`.

### Ошибка 3: путать адрес и значение
```assembly
mov rax, msg       ; rax = адрес
mov rax, [msg]     ; rax = первые 8 байт строки
```

### Ошибка 4: неверный расчёт смещения в массиве
```assembly
arr dd 10, 20, 30
; arr[1] — это arr + 4, не arr + 1!
; arr[2] — это arr + 8
```
Для `dd` каждый элемент = 4 байта. Для `dw` = 2 байта. Для `dq` = 8 байт.

## 12. Упражнения

1. **Одна переменная.** Объяви в .data переменную `x dd 42`. Прочитай её в RAX. Выйди с кодом возврата = x.

2. **Массив.** Объяви массив `arr dd 7, 14, 21, 28, 35`. Прочитай arr[3] (индекс 3) и выйди с этим кодом. Убедись, что получилось 28.

3. **Строка.** Объяви строку "Xodex" в .data. Выведи её через write.

4. **Два массива.** Объяви два массива по 3 элемента. Сложи arr1[0] + arr2[0], результат — код возврата.

5. **Запись.** Объяви `result dq 0`. Запиши туда число 123456. Потом прочитай и выйди с ним.
