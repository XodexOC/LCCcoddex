# [01] Первая программа: syscall

> **Трек:** Assembly · **Уровень:** 01 · **Сложность:** ★☆☆☆☆

## 1. Какую проблему решаем?

Мы хотим, чтобы компьютер **что-то сделал**: завершился с кодом 0, напечатал строчку.

Но процессор не умеет «завершиться» или «напечатать». Любое взаимодействие с внешним миром делает **ядро операционной системы** (Linux) через **системный вызов** — `syscall`.

## 2. Структура программы

```assembly
section .text       ; секция кода
    global _start   ; точка входа для линкера

_start:             ; метка (label)
    ...инструкции... ; что будет выполнять процессор
```

**section .text** — секция кода (машинные инструкции).

**global _start** — делает метку `_start` видимой для линкера.

**_start:** — метка. Имя для адреса в памяти. С неё начинается программа.

## 3. Первая программа: exit(0)

```assembly
; exit.asm — завершиться с кодом 0
section .text
    global _start

_start:
    mov rax, 60     ; rax = 60 (номер syscall exit)
    xor rdi, rdi    ; rdi = 0 (код возврата)
    syscall         ; передать управление ядру
```

**mov rax, 60** — положить число 60 в регистр RAX.

**xor rdi, rdi** — положить 0 в rdi (xor числа с самим собой = 0).

**syscall** — передать управление ядру. Ядро смотрит rax (60 = exit), берёт аргумент из rdi (0 = успех).

### Номера syscall (Linux x86-64)

| Номер | Имя | Что делает |
|-------|-----|-----------|
| 0 | read | Читает из файла |
| 1 | write | Пишет в файл |
| 2 | open | Открывает файл |
| 60 | exit | Завершает программу |

Аргументы передаются по порядку: rdi → rsi → rdx → r10 → r8 → r9.

## 4. Как собрать и запустить

```bash
nasm -f elf64 exit.asm -o exit.o
ld exit.o -o exit
./exit
echo $?    → 0
```

## 5. Вторая программа: hello world

```assembly
; hello.asm — вывести "Hello!" и завершиться
section .data
    msg db "Hello!", 10
    len equ $ - msg

section .text
    global _start

_start:
    mov rax, 1          ; write
    mov rdi, 1          ; stdout
    mov rsi, msg        ; адрес строки
    mov rdx, len        ; длина
    syscall

    mov rax, 60         ; exit
    xor rdi, rdi        ; 0
    syscall
```

### Что такое `msg db "Hello!", 10`?

| Часть | Что значит |
|-------|-----------|
| `msg` | **Метка** — имя для адреса первого байта строки |
| `db` | **Define Byte** — выделить байты с указанными значениями |
| `"Hello!"` | 6 символов, каждый превратится в ASCII-код (1 байт) |
| `, 10` | Ещё один байт — код перевода строки (`\n`) |

Итого в памяти:
```
msg → [72] [101] [108] [108] [111] [33] [10]
       H     e     l     l     o     !    \n
```

### Что такое `len equ $ - msg`?

| Часть | Что значит |
|-------|-----------|
| `len` | Имя константы |
| `equ` | **Equate** — вычислить и запомнить число |
| `$` | Текущий адрес (сразу после строки) |
| `msg` | Адрес начала строки |
| `$ - msg` | Разница = длина строки в байтах |

Автоматический подсчёт длины. Если изменить строку — `len` пересчитается сам.

### Собираем и запускаем

```bash
nasm -f elf64 hello.asm -o hello.o
ld hello.o -o hello
./hello
```
Вывод: `Hello!`

## 6. Частые ошибки

- **nasm: command not found** — установи: `sudo apt install nasm`
- **undefined reference to _start** — забыл `global _start` или метка называется иначе
- **Segmentation fault** — забыл exit в конце программы
- **Ничего не выводится** — неверная длина строки, проверь `len equ $ - msg`
