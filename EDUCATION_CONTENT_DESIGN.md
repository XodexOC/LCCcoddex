# Xodex Education Content Design

## 1. Philosophy
Xodex is not a course platform. Materials should be:
- Reference guides
- Exploration maps
- Knowledge libraries

Users can study topics in any order.

## 2. Knowledge Areas

### Computer Fundamentals
Topics:
- Binary systems
- Computer architecture
- CPU operation
- Memory hierarchy
- Storage systems
- Operating systems

### UNIX/Linux
Topics:
- Filesystem structure
- Process management
- Permissions model
- Shell scripting
- Kernel internals
- Networking basics

### Programming Languages
Languages:
- C
- C++
- Python
- Rust
- JavaScript
- SQL
- Assembly

### Engineering
Topics:
- Algorithms
- Database systems
- System architecture
- Debugging techniques
- Security principles

## 3. Content Structure
```
/modules/c/
├── docs/         # Documentation
├── examples/     # Code examples
├── projects/     # Practical projects
└── references/   # External references
```

Each topic includes:
- Theoretical explanation
- Practical examples
- Experiment guides
- Reference materials

## 4. Learning Depth Model

### Level 0: Introduction
- What is it?
- Basic concepts

### Level 1: User Level
- How to use it?
- Practical applications

### Level 2: Developer Level
- How does it work internally?
- Implementation details

### Level 3: Engineer Level
- Why was it designed this way?
- Architectural decisions

### Level 4: Researcher Level
- Source code analysis
- Standards compliance
- System architecture

## 5. Example Philosophy
Examples must:
- Be small and focused
- Illustrate single concepts
- Be executable by users
- Include clear comments

Avoid:
- Large unexplained projects
- Code copying without understanding

## 6. Practical Experiments
Experiment format:
```
Topic: Memory

Theory:
How memory works in computers

Experiment:
Create a program demonstrating:
- Stack allocation
- Heap allocation
- Memory mapping

Observation:
Analyze using:
- gdb
- strace
- /proc filesystem

Conclusion:
Understand memory behavior
```

## 7. Project Based Learning

### Beginner Projects
- Calculator
- Text editor
- File manager

### Intermediate Projects
- HTTP server
- Database engine
- Compiler

### Advanced Projects
- Custom shell
- Mini operating system
- Kernel module development

## 8. Documentation Quality Rules
Materials must:
1. Explain reasons behind concepts
2. Include practical exercises
3. Provide reference links
4. Be available in RU/EN
5. Show progression paths

## 9. External Knowledge
Integration with:
- Official documentation
- Reference books
- Industry standards
- Source code repositories

## 10. Content Rules
1. No checkbox learning
2. No artificial progression
3. Quality over quantity
4. Understanding over memorization
