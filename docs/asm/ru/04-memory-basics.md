# [04] Память: секция .data

> **Трек:** Assembly · **Уровень:** 04 · **Сложность:** ★★☆☆☆

## 1. Какую проблему решаем?

Регистров мало — всего 16, каждый хранит 8 байт. Для строк, массивов и структур нужна **память**. Секция .data — область памяти, где можно объявить переменные любого размера.

## 2. Что такое секция .data?

```assembly
section .data
    msg db "Hello!", 10
```

Ассемблер:
1. Выделяет место в исполняемом файле
2. Заполняет байтами: 72 101 108 108 111 33 10
3. Запоминает адрес как `msg`

## 3. Директивы

| Директива | Размер | Пример |
|-----------|--------|--------|
| `db` | 1 байт | `x db 10` |
| `dw` | 2 байта | `x dw 1000` |
| `dd` | 4 байта | `x dd 100000` |
| `dq` | 8 байт | `x dq 123456789` |
| `resb` | 1 байт × N | `buf resb 256` |
| `resd` | 4 байта × N | `arr resd 100` |

### db — Define Byte (1 байт)
```assembly
byte_var db 42                 ; 1 байт
arr db 10, 20, 30, 40         ; 4 байта
str db "Hello", 0             ; 6 байт
```

### dw — Define Word (2 байта)
```assembly
w dw 1000                     ; 2 байта: E8 03 (little-endian)
```

### dd — Define Double word (4 байта)
```assembly
d dd 100000                   ; 4 байта
```

### dq — Define Quad word (8 байт)
```assembly
q dq 123456789                ; 8 байт
```

### resb/resw/resd/resq — резервирование
```assembly
buffer resb 256               ; 256 байт (не заполнены)
array  resd 100               ; 400 байт
```

## 4. Разница между меткой и значением

```assembly
section .data
    my_num dq 42

section .text
    mov rax, my_num       ; rax = АДРЕС (например, 0x601000)
    mov rbx, [my_num]     ; rbx = ЗНАЧЕНИЕ (42)
    mov [my_num], rcx     ; записать rcx в my_num
```

**my_num** — адрес (как номер дома на улице).
**[my_num]** — значение по этому адресу (что внутри дома).

## 5. Массивы

```assembly
section .data
    arr dd 10, 20, 30, 40, 50
```

Обращение к элементу:
```assembly
mov eax, [arr]        ; arr[0] = 10
mov eax, [arr + 4]    ; arr[1] = 20 (каждый dd = 4 байта)
mov eax, [arr + 8]    ; arr[2] = 30
mov eax, [arr + 12]   ; arr[3] = 40
```

**Смещение = индекс × размер_элемента.**

## 6. Вычисление длины: equ и $

```assembly
section .data
    msg db "Hello!", 10
    len equ $ - msg
```

- `msg` — адрес начала строки
- `$` — адрес сразу после строки
- `$ - msg` = длина в байтах
- `equ` — вычислить и запомнить, не выделяет память

## 7. Запись в память

```assembly
mov dword [var], 100    ; записать 4 байта
mov qword [var], 100    ; записать 8 байт
mov byte [var], 10      ; записать 1 байт
```

**Важно:** указывать размер при записи в память!

## 8. Программа

```assembly
section .data
    msg db "Array[2] = 30", 10
    arr dd 10, 20, 30, 40, 50

section .text
    global _start

_start:
    ; читаем arr[2]
    mov eax, [arr + 8]
    ; eax = 30

    ; выводим строку
    mov rax, 1
    mov rdi, 1
    mov rsi, msg
    mov rdx, 15
    syscall

    ; exit(0)
    mov rax, 60
    xor rdi, rdi
    syscall
```

## 9. Частые ошибки

- **Забыть размер при записи** — `mov [var], 5` не скомпилируется. Пиши `mov dword [var], 5`
- **Читать больше, чем объявлено** — `mov rax, [b]` где b db 10 — прочитает 8 байт, 7 — мусор
- **Путать адрес и значение** — `mov rax, msg` vs `mov rax, [msg]`
- **Неверное смещение** — `arr dd` → каждый элемент 4 байта, arr[2] = arr + 8, не arr + 2

## 10. Упражнения

2. Объяви `arr dd 7, 14, 21, 28, 35`. Прочитай arr[3], выйди с результатом (должно быть 28).
3. Объяви строку "Xodex" в .data. Выведи через write.
4. Объяви `result dq 0`. Запиши 123456, прочитай, выйди с ним.
